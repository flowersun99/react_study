react.js는 node.js 기반으로 돌아가기에 nodejs를 알아야 함

Nodejs는 Javascript의 실행 환경(Run time) == 구동기

구동기란?
구동하게 도와주는 기계 ex. 게임을 실행시켜주는 닌텐도 같은 역할

Node가 만들어진 이유는 뭘까?
==> 원래는 js가 웹 페이지 내부에 필요한 기능을 만들기 위해 만들어진 언어
오직 웹 브라우저 내에서만 작동할 수 있도록 만들었기에 유연하고 생산성이 높았음

그런데 자바스크립트 좋아하는 사람들이 많아져서 웹 브라우저 밖에서도 js 쓰게 만들고 싶어짐.
그래서 node js를 만들어서 웹 서버, 모바일 앱, 데스크톱 앱, ... 


프로젝트 :: 특정 목적을 갖는 프로그램의 단위
패키지 :: Node.js에서 사용하는 프로그램의 단위

package.json에서 start를 통해서 root file 지정하고 npm start하면 바로 실행시켜 줌

모듈 시스템 :: 모듈을 다루는 시스템
모듈이란? 

보통 기능을 구현한다 하면 각각의 파일을 만들어서 구현 => 생산성 향상 목적
각각의 기능을 맡는 것을 모듈이라고 하고 이 모듈을 관리하는 것이 모듈 시스템임

"type": "module"을 package.json에 넣으면 es모듈 시스템을 사용하는 것임
React에서 이거 많이 씀

라이브러리란? 프로그램을 개발할 때 필요한 다양한 기능들을 미리 만들어 모듈화 해놓은 것
